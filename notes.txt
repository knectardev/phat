## 3. Implementation Blueprint
You can add a `useBacktest` hook to your `App` component that maintains its own state, separate from the feed.

```javascript
// Implementation Concept for useBacktest.js
const useBacktest = (currentBar, signals, settings) => {
  const [account, setAccount] = useState({ balance: 10000, position: null, history: [] });

  useEffect(() => {
    if (!currentBar) return;

    // 1. Check Exit Conditions (SL/TP)
    if (account.position) {
      handlePositionManagement(currentBar, account, setAccount);
    }

    // 2. Check Entry Conditions (Decoupled from Feed Logic)
    const { trend, reversal } = signals;
    if (!account.position && trend.tier === 'HIGH' && reversal.score < 0.3) {
       executeEntry('LONG', currentBar.close);
    }
  }, [currentBar]); // Only triggers when a new bar is "emitted"
};
```

## 5. Avoiding the "Observer Effect" (Non-Cheating)
To ensure the algorithm is truly decoupled:
* **Latency Simulation:** Force the algorithm to execute its trade on the *Open* of the next bar, rather than the *Close* of the bar that generated the signal.
* **Slippage:** Use your `simulatedSpread` (REQ-FEED-02) to penalize every entry and exit. This prevents the "perfect math" trap where the bot wins simply because it's trading on theoretical mid-prices.
* **No Phase Awareness:** Ensure the trading hook does not have access to the `PHASES` array or the `targetQuadrant` hint from the generator.
